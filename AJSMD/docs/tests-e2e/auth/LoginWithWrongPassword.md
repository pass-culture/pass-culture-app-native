# Test de connexion échoué

Ce document décrit le processus suivi par un utilisateur lorsqu'il tente de se connecter à l'application, mais que la connexion échoue.

## 1. Démarrage de la connexion

*   L'utilisateur appuie sur le bouton "Se connecter".

## 2. Saisie des informations de connexion

*   L'utilisateur appuie sur le champ "Email".
*   Un autre parcours (non détaillé ici) est exécuté pour saisir une adresse email.
*   L'utilisateur appuie sur le champ "Mot de passe".
*   L'utilisateur saisit un mot de passe aléatoire.
*   L'utilisateur ferme le clavier.
*   L'utilisateur appuie sur le bouton "Se connecter".

## 3. Vérification de l'échec de la connexion

*   L'application affiche un message d'erreur indiquant que l'e-mail ou le mot de passe est incorrect.
*   L'utilisateur appuie sur le bouton "Revenir en arrière" pour retourner à la page de connexion et potentiellement réessayer.
